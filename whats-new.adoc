==== EVS TCO 计算器中的 i7i.metal-24xl 实例支持

EVS TCO 计算器现在支持 AWS i7i.metal-24xl EC2 实例作为集群主机。与 i4i.metal 配置相比，这些实例将计算的 EVS 起始成本降低了约 33%，将 vSAN 存储的 EVS 起始成本降低了约 25%。

==== 通过同步延迟、IOPS 和吞吐量图表增强延迟分析

延迟分析面板现在包括一个具有同步性能图表的*详细延迟分析*页面：

* *延迟图*：显示延迟如何随时间变化，以及超出限制时的阈值线和警报。
* *IOPS 图形*：显示每秒操作数，以及过去超出限制的阈值线和指示器。
* *吞吐量图*：显示数据传输速率（DataReadBytes 或 DataWriteBytes）。在筛选器中选择元数据时，吞吐量图表不显示任何数据。


所有图形都使用相同的时间线，并根据事件是读取还是写入自动显示正确的指标。

您可以调整视图，并将延迟峰值与 IOPS 和吞吐量的变化相关联。违规标记和悬停详细信息可帮助您查看超出阈值的时间，并查看检测到的违规行为的关键指标。

https://docs.netapp.com/us-en/workload-eda/task-analyze-latency-trends.html["了解如何使用多个图表分析延迟趋势"^]。



==== 设置延迟通知频率

现在，您可以选择接收延迟违规电子邮件通知的频率：

* *每天提醒我（推荐）*：当任何 FSx for ONTAP 文件系统出现新问题时，您将收到通知。


如果同一个文件系统在一天内出现多个延迟问题，您只会收到一封电子邮件。

在发送文件系统的延迟警报后，Workload Factory 等待 24 小时后再为该文件系统发送另一个警报。

* *任何延迟违规通知（每 20 分钟一次）*：检测到违规时收到通知，最多每 20 分钟一次。


默认情况下，每日通知处于启用状态，以减少电子邮件数量，同时仍提醒您注意延迟问题。您可以在设置延迟监控或编辑阈值设置时更改通知首选项。

https://docs.netapp.com/us-en/workload-eda/task-configure-latency-monitoring.html["了解如何配置延迟监控"^]。

==== 在 NetApp Workload Factory 中跟踪复制延迟

使用 NetApp Workload Factory，您可以跟踪配置了远程复制的 FSx for ONTAP 文件系统的复制延迟。复制延迟表示数据写入源文件系统和复制到目标文件系统之间的时间延迟。跟踪复制延迟可帮助您监控复制设置的运行状况，并确定可能影响数据一致性和恢复点目标 (RPO) 的潜在问题。

link:https://docs.netapp.com/us-en/workload-fsx-ontap/track-replication-delays.html["在 NetApp Workload Factory 中跟踪复制延迟"]



==== NetApp Workload Factory 中的主机标识和工作负载定义

Workload Factory 标识使用存储的主机以及与主机活动相关联的工作负载类型。工作负载定义是基于 AWS 标记和元数据的 AI 生成的。您可以从 Workload Factory 中的文件系统、卷和块设备资源屏幕查看主机详细信息和工作负载定义。



==== NetApp Workload Factory 中可用的 LIF 详细信息

我们直接在文件系统的"概述"选项卡中显示所有文件系统和存储虚拟机端点。Workload Factory 控制台内的直接端点可见性使您可以更轻松地监控网络访问、端点和故障转移状态，而无需使用 CLI 或 CloudShell。要显示 LIF 详细信息，包括角色、端点、节点、IP 地址和状态，您需要将链接与文件系统关联。

link:https://docs.netapp.com/us-en/workload-fsx-ontap/create-link.html["了解如何在 NetApp Workload Factory 中将链接与您的文件系统关联"]。



==== 成本增强功能

新的可选页面级筛选器可用于改进所有 AWS 账户的资源成本分析结果。您可以选择多个区域，并使用文件系统分配标记键和键值来过滤成本分析结果，并获取针对 FSx for ONTAP 文件系统和卷的资源特定成本详细信息。

link:https://docs.netapp.com/us-en/workload-fsx-ontap/track-costs.html["在 NetApp Workload Factory 中使用筛选器进行成本分析"]



==== NetApp Workload Factory 中提供复制初始化调度

在数据复制设置期间，您现在可以为 FSx for ONTAP 文件系统安排复制初始化。选项是立即或在指定时间初始化复制关系。复制初始化是通过将数据从源文件系统复制到目标文件系统来创建复制基线的过程。计划复制初始化允许您自动执行此过程，并确保您的复制设置为最新，而无需手动干预。

link:https://docs.netapp.com/us-en/workload-fsx-ontap/create-replication.html["在 NetApp Workload Factory 中复制数据"]



==== FSx for ONTAP 文件系统的良好架构状态的 Excel 报告

分析后，您可以下载 FSx for ONTAP 文件系统的良好架构状态的 Excel 报告，用于记录或与您的团队共享。

link:https://docs.netapp.com/us-en/workload-fsx-ontap/improve-configurations.html["在 NetApp Workload Factory 中查看 FSx for ONTAP 文件系统的良好架构状态"]



==== 查看和分析延迟事件

Workload Factory 现在支持查看和分析 FSx for ONTAP 卷的延迟事件。您可以在没有链接连接的情况下查看延迟事件，从而更轻松地监控性能问题。要分析延迟事件并接收基于 AI 的见解和建议，您必须使用链接连接到文件系统。分析提供当前延迟值、阈值、QoS 响应分析和补救指导。

link:https://docs.netapp.com/us-en/workload-fsx-ontap/analyze-events.html["在 NetApp Workload Factory 中分析 FSx for ONTAP 事件"]



==== 设置卷延迟阈值

现在，您可以按卷配置延迟阈值，以便在读取或写入延迟超过定义的限制时监控性能并接收警报。延迟监控可帮助您在影响工作负载之前主动识别性能问题。

link:https://docs.netapp.com/us-en/workload-fsx-ontap/set-volume-latency-thresholds.html["在 NetApp Workload Factory 中设置卷延迟阈值"]

编辑对话框根据您的选择自动填充：如果所有选定的文件系统共享相同的参数值，则会显示这些值；否则，将使用默认值。这有助于您在整个环境中应用一致的容量设置。

* *建议决定点*：悬停以查看时间戳、建议增加的大小和当前容量。
* *自动化决策点*：悬停以查看时间戳、成功或失败状态、增加大小和新的聚合大小。失败的尝试包括失败原因。


决策点可以每 30 分钟发生一次，并根据选定的时间范围单独显示或合并显示。

link:https://docs.netapp.com/us-en/workload-eda/task-use-overview-dashboard.html#analyze-capacity-trends-and-decision-points["了解容量决策点"^]。

==== 延迟事件的电子邮件和 SNS 通知

现在，当在 FSx for ONTAP 卷上检测到延迟事件时，您可以接收自动通知。在 Workload Factory 设置中配置电子邮件或 Amazon SNS 通知，以便在卷违反配置的延迟阈值时接收实时警报。

通知按文件系统发送，包括有关所有受影响卷、严重性级别、阈值比较的详细信息，以及直接链接到 Latency 页面进行调查。

https://docs.netapp.com/us-en/workload-eda/task-monitor-latency.html#configure-latency-notifications["了解如何配置延迟通知"^]。



==== 使用交互式图形实现延迟趋势可视化

延迟分析面板现在包括一个选项卡式界面，用于全面的延迟分析：

* *概述*：查看自动基本分析结果，显示哪个组件导致延迟
* *随着时间的推移*：使用跨多个时间范围（1 小时、3 小时、12 小时、24 小时或 72 小时）的交互式图表可视化卷延迟行为


*随着时间的推移*选项卡显示原始 CloudWatch 延迟数据（读取或写入，具体取决于触发的警报），以及警告和关键阈值线、违规指标和每个违规行为的详细指标。

这种全面的可视化可帮助您识别模式，了解延迟问题是重复出现还是孤立存在，并就补救措施做出更明智的决策。

https://docs.netapp.com/us-en/workload-eda/task-analyze-latency-trends.html["了解如何分析延迟趋势"^]。
